# Whack-O-Meter — Session Log

Tidy summary of the Cursor agent session that built and refined this project.

---

## Goal

Build a browser-based app (**Whack-O-Meter**) that:

- Graphs force sensor data from CSV files in `raw_data/`
- Uses a cassette-futurism / cyberpunk cathode aesthetic
- Auto-trims significant portions of force/time readings
- Provides axis scale sliders and CSV search/navigation
- Runs AI analysis with web research, stored in `analysis/memory.md`
- Hosts on GitHub Pages

---

## What Was Built

### Frontend (Vite + React + TypeScript)

| Feature | Implementation |
|---------|----------------|
| UI theme | CRT scanlines, phosphor green/magenta, `VT323` + `Share Tech Mono` fonts |
| Graphs | uPlot force vs. time charts |
| CSV parsing | Flexible header detection (`time`, `force`, `g`, `accel`, etc.) |
| Auto-trim | Baseline + threshold algorithm in `src/lib/autoTrim.ts` |
| Navigation | Search bar + prev/next buttons to cycle CSV files |
| Analysis panel | Renders markdown from `analysis/memory.md` |

### Data & Build

- CSV manifest generated by `scripts/generate_manifest.js`
- Sample CSVs added for testing; repo later populated with real force-meter exports
- CSVs loaded at runtime via `raw.githubusercontent.com` (not bundled into Pages deploy)

### AI Pipeline (GitHub Actions)

- `analyze.yml` — analyzes CSV curves, searches sports-science/concussion literature, updates `analysis/memory.md`
- `deploy.yml` — builds and deploys the SPA to GitHub Pages

User configured a Gemini API key in repository secrets.

### 3. Model set to gemini-3.1-flash

User requested **`gemini-3.1-flash`** as the analysis model.

**Changes:**

- Default model updated in `scripts/analyze_csv.py`
- `GEMINI_MODEL: gemini-3.1-flash` set in analyze workflow

### 4. Live site 404

User reported the live site was 404ing.

**Diagnosis:**

- Deploy workflow **build succeeded** but **deploy failed**
- GitHub Pages was **not enabled** on the repository
- Base path was wrong (`/whack-o-meter/` vs repo name `/Whack-O-Meter/`)
- Build was copying ~91 large CSV files into the deploy artifact

**Fixes (PR #3):**

- Document one-time Pages setup: **Settings → Pages → Source: GitHub Actions**
- Derive Vite `base` from repo name (`/Whack-O-Meter/`)
- Stop bundling CSVs; load via `rawUrl` in manifest
- Correct live URL in README

  ├── raw_data/         → force meter CSV exports
